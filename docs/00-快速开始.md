# 00-快速开始

## 适用范围

本页只描述当前仓库已经存在并可验证的构建、安装、运行与最小验收路径。

## 前置条件

- 编译器：支持 C++23
- CMake：`>= 3.16`
- 第三方头文件：需要能找到 `<concurrentqueue/moodycamel/*.h>`
- Linux：
  - `DISABLE_IOURING=ON` 时使用 epoll；若要异步文件 IO，需要 `libaio`
  - `DISABLE_IOURING=OFF` 且系统存在 `liburing` 时才会切到 io_uring
- 命名模块：
  - 仅在 `CMake >= 3.28`
  - 且生成器支持模块（本仓库当前接受 Ninja / Visual Studio）
  - 且编译器不是 AppleClang

## 默认构建

```bash
cmake -S . -B build -DBUILD_TESTS=ON -DBUILD_EXAMPLES=ON -DBUILD_BENCHMARKS=ON
cmake --build build --parallel
```

## 安装与消费

最小安装命令：

```bash
cmake --install build-docverify --prefix "$PWD/build-docverify/install"
```

本轮已验证的 `find_package` 烟雾测试：

```bash
cmake -S test/package-consumer -B build-docverify/package-consumer \
  -DCMAKE_PREFIX_PATH="$PWD/build-docverify/install"
cmake --build build-docverify/package-consumer --parallel
./build-docverify/package-consumer/galay_consumer
```

当前结果：

- `find_package(galay-kernel CONFIG REQUIRED)`：配置成功
- `target_link_libraries(... galay-kernel::galay-kernel)`：构建成功
- `GALAY_KERNEL_SUPPORTED_HEADERS` / `GALAY_KERNEL_INTERNAL_HEADERS`：可读取
- 运行结果：`consumer exit=0`

说明：

- `test/package-consumer/` 是仓库内受版本控制的最小 consumer fixture，不再依赖 build 目录中的临时 `consumer-src`
- 安装包同时导出 `GALAY_KERNEL_PACKAGE_CONSUMER_FIXTURE_DIR`，供安装树验证时复用同一份 fixture

完整企业接入说明见 `docs/21-企业接入与验证.md`。

## 当前已知限制

- 仓库当前没有 `ENABLE_LOG` 选项，也没有 `spdlog` 依赖链路
- `MpscChannel<T>` 没有 `close()`；发送端是同步 `bool send(...)`
- `HandleOption` 当前只公开 `handleBlock()`、`handleNonBlock()`、`handleReuseAddr()`、`handleReusePort()`
- `AioFile` 只在 `USE_EPOLL` 下公开
- benchmark 数字是当前机器单次运行结果，不是跨平台性能承诺
