# 03-使用指南

本页只给出当前公开头文件、真实示例、真实测试里可落地的工作流。每个条目都标注来源文件、target、运行命令、环境变量与执行模式，避免再出现“代码片段能看不能跑”的漂移。

## 通用前提

- 编译器：支持 C++23
- CMake：`>= 3.16`
- 头文件依赖：可解析 `<concurrentqueue/moodycamel/*.h>`
- 默认构建：

```bash
cmake -S . -B build -DBUILD_TESTS=ON -DBUILD_EXAMPLES=ON -DBUILD_BENCHMARKS=ON
cmake --build build --parallel
```

## 1. 启动 Runtime 并获取调度器

- 真实头文件：`galay-kernel/kernel/Runtime.h`
- 真实验证：`test/T27-runtime_stress.cc`
- 真实验证：`test/T42-runtime_strict_scheduler_counts.cc`
- target：`T27-runtime_stress`、`T42-runtime_strict_scheduler_counts`
- 运行命令：

```bash
cmake --build build --target T27-runtime_stress T42-runtime_strict_scheduler_counts
./build/bin/T27-runtime_stress
./build/bin/T42-runtime_strict_scheduler_counts
```

- 环境变量：无
- 执行模式：自测型测试 target

最小用法：

```cpp
Runtime runtime = RuntimeBuilder()
    .ioSchedulerCount(2)
    .computeSchedulerCount(2)
    .build();

runtime.start();

IOScheduler* io = runtime.getNextIOScheduler();
ComputeScheduler* compute = runtime.getNextComputeScheduler();

runtime.stop();
```

注意：

- `Runtime::start()` 会自动启动全局 `TimerScheduler`
- 访问调度器请使用 `getIOScheduler(size_t)` / `getComputeScheduler(size_t)` 或 `getNext*()`
- `0` 表示禁用对应默认调度器；自动数量请使用 `GALAY_RUNTIME_SCHEDULER_COUNT_AUTO`

## 2. 运行一个 TCP Echo 自闭环示例

- 源文件：`examples/include/E2-tcp_echo_server.cc`
- target：`E2-TcpEchoServer`
- 运行命令：

```bash
cmake --build build --target E2-TcpEchoServer
./build/bin/E2-TcpEchoServer
```

- 环境变量：无
- 执行模式：自闭环示例；程序内部同时拉起服务端和客户端
- 固定端口：`127.0.0.1:9082`

关键 API：

```cpp
TcpSocket listener;
listener.option().handleReuseAddr();
listener.option().handleNonBlock();

listener.bind(Host(IPType::IPV4, "127.0.0.1", 9082));
listener.listen(16);

Host peer;
auto accepted = co_await listener.accept(&peer);
```

## 3. 运行一个 TCP Client 自闭环示例

- 源文件：`examples/include/E3-tcp_client.cc`
- target：`E3-TcpClient`
- 运行命令：

```bash
cmake --build build --target E3-TcpClient
./build/bin/E3-TcpClient
```

- 环境变量：无
- 执行模式：自闭环示例；程序内部自带回包服务端，客户端走真实 `connect/send/recv`
- 固定端口：`127.0.0.1:9083`

关键 API：

```cpp
TcpSocket socket;
socket.option().handleNonBlock();

auto connected = co_await socket.connect(Host(IPType::IPV4, "127.0.0.1", 9083));
if (!connected) {
    co_return;
}

auto sent = co_await socket.send(message, message_len);
auto recv = co_await socket.recv(buffer, sizeof(buffer));
```

最小模式：

```cpp
AsyncMutex mutex;

Coroutine worker() {
```

## 6. 使用 MpscChannel

- 真实测试：`test/T14-mpsc_channel.cc`
- 真实 import 示例：`examples/import/E6-mpsc_channel.cc`
- 当前可运行 target：`T14-mpsc_channel`
- 运行命令：

```bash
cmake --build build --target T14-mpsc_channel
./build/bin/T14-mpsc_channel
```

- 环境变量：无
- 执行模式：测试驱动示例；import 版本取决于模块 target 是否生成

最小模式：

```cpp
MpscChannel<int> channel;

channel.send(42);
auto value = co_await channel.recv();
```

注意：

- `send(...)` 是同步 `bool` 返回，不是 `co_await channel.send(...)`
- 当前没有 `channel.close()`
- 结束消费时应使用业务协议、计数、哨兵值或额外同步原语

## 7. 使用 FileWatcher

- 真实头文件：`galay-kernel/async/FileWatcher.h`
- 真实测试：`test/T9-file_watcher.cc`
- 深页：`docs/19-文件监控.md`
- target：`T9-file_watcher`
- 运行命令：

```bash
cmake --build build --target T9-file_watcher
./build/bin/T9-file_watcher
```

- 环境变量：无
- 执行模式：测试 target；会自行创建与修改临时文件

最小模式：

```cpp
FileWatcher watcher;

auto wd = watcher.addWatch("/tmp/app.conf", FileWatchEvent::Modify);
if (!wd) {
    co_return;
}

auto event = co_await watcher.watch();
```

注意：

- 构造函数无参数
- 真实接口是 `addWatch(...)` + `watch()`
- 目录监控与文件名回传的行为在 Linux 与 macOS 上存在差异
