## 当前结果：`B8-MpscChannel`

代表性输出：

- 延迟测试：`100000` 条消息，平均延迟 `1392.88 us`
- 跨调度器测试：`1000000` 条消息，`68 ms`，吞吐约 `14.7M msg/s`
- 持续负载测试：总发送 / 接收 `65319505` 条，平均吞吐约 `11.4M msg/s`
- 正确性测试：`no_loss=YES`、`no_duplicate=YES`、`all_correct=YES`

解释：

- 这是当前机器、当前构建参数、单次运行结果
- 该 benchmark 同时覆盖延迟、正确性、跨调度器吞吐和持续负载

## 当前结果：`B10-Ringbuffer`

代表性输出：

- Wrap around：`17433.8 MB/s`
- `getWriteIovecs`：`1.85890 ns/call`
- `getReadIovecs`：`1.07673 ns/call`
- `produce+consume`：`8.03958 ns/pair`
- Network receive simulation：`72400.8 MB/s`
- Network send simulation：`53022.3 MB/s`

解释：

- 这些数字来自当前机器单次运行
- `RingBuffer` 对缓存、CPU 频率和编译器非常敏感，不能直接外推到其他平台

## 当前结果：`B13-Sendfile`

| 文件大小 | sendfile | read+send | 当前观察 |
|---|---|---|---|
| 1 MB | `73 ms / 13.6986 MB/s` | `71 ms / 14.0845 MB/s` | 小文件下 `read+send` 略快 |
| 10 MB | `73 ms / 136.986 MB/s` | `70 ms / 142.857 MB/s` | 小中型文件差距不稳定 |
| 50 MB | `66 ms / 757.576 MB/s` | `86 ms / 581.395 MB/s` | `sendfile` 更快 |
| 100 MB | `98 ms / 1020.41 MB/s` | `107 ms / 934.579 MB/s` | `sendfile` 更快 |

注意：

- 这是本机 loopback + 当前文件缓存条件下的结果
- benchmark 输出中 1MB / 10MB 组出现负提升，说明小文件路径高度受缓存与调度波动影响
- 这些数字不应被当作跨平台 SLA 或稳定承诺

## 复现实务建议

- 固定构建类型、编译器与后端
- 记录 CPU、内核 / 系统版本、是否在容器、是否启用页缓存预热
- 网络类 benchmark 需要说明 loopback / 实网 / 容器环境
- 文件类 benchmark 需要说明测试文件大小和缓存状态
- 如果问题先是“接口怎么用”，先回到 `docs/02-API参考.md` 与 `docs/03-使用指南.md`，不要直接从补充页或历史数字出发
