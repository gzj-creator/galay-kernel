# 11-协程

本页现在只保留 `Coroutine` / `spawn` / `sleep` 相关专题定位；完整语义请优先回到主干页。

## 本页回答什么

- 仓库里“协程”的公开入口是什么
- `spawn(...)`、`wait()`、`sleep(...)` 的问题应先看哪里
- 哪些测试 / 示例最适合作为语义锚点

## 当前稳定事实

- 基本协程类型是 `Coroutine`
- 链式调度、等待另一个协程、睡眠等能力都已回流到主干页说明
- `sleep(...)` 依赖运行时中的全局 `TimerScheduler`；涉及时间系统时不要只看本页，要连同主干页一起看
