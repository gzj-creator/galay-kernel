本页现在只保留网络 IO 专题的定位信息；公开 API、用法与边界请优先回到主干页。

## 本页回答什么

- `TcpSocket`、`UdpSocket`、`HandleOption`、`Host` 的问题该先看哪里
- `readv/writev`、`sendfile` 和网络数据面的边界是什么
- 哪些源码 / 测试 / benchmark 是可靠锚点

## 当前稳定事实

- `TcpSocket` / `UdpSocket` / `HandleOption` / `Host` 的公开边界以 `docs/02-API参考.md` 为准
- `readv/writev` 与 `sendfile` 已被折回 `docs/03-使用指南.md`、`docs/05-性能测试.md`、`docs/06-高级主题.md`
- Linux `io_uring` 后端下，plain `recv` 现在走 `multishot recv + provided buffer ring + controller ready queue`：
  - reactor 维护内部 provided buffer ring
  - CQE 先落到 `IOController` 的 staged recv queue
  - awaitable 恢复时再拷贝到用户传入 buffer
  - 终态 CQE 后会重新挂载 multishot recv
- 如果问题还是“接口怎么调用”，不要先从本页开始，应先看主干页
